1. 显示过滤/排序后的结果

如果我们想要显示一个经过过滤或者排序后的数组，不更改或者重置原始数组数据，这种情况下我们有两种方案，第一种，通过计算属性来返回过滤后的结果直接 v-for：

  • {{ n }}

第二种，通过 v-for 遍历一个方法，将原数组作为参数传入（方法仅会调用一次）：

  • {{ n }}

4. v-for 与 v-if 一同使用

在 v-if 时也简单提到了，不推荐在同一个元素上一起使用，在此我们展开讲讲为什么，顺便演示一下：

当它们处于同一节点时，v-for 的优先级要高于 v-if，意味着 v-if 将分别重复运行在每个 v-for 遍历的节点中，如果你的需求是这样，有一个 todo 列表，你想只渲染未完成的 todo 信息，可以这样做：

  • 未完成的任务：{{ todo.message }}

显然，这并不够优雅，因为，哪怕是只需要渲染一小部分的数据时，同样需要遍历整个列表，不管列表内的数据是否发生了改变，出现这样的需求时，完全可以把这个操作抽离到一个计算属性中过滤后使用，这样的好处是：

  • 过滤后的列表比原先列表数据更少，渲染更快
  • 只会在数组发生变更后才重新计算，更加高效
  • 解耦渲染层逻辑，可维护性（扩展性）更强

5. 在组件上使用 v-for

在自定义组件上使用，与普通元素使用一样无异，但需要注意的是：

  1. Vue2.2.0+ 版本，在组件上使用 v-for 是，必须要指定 key 值，否则会有 Vue tip
  2. 数据不会自动注入到组件内部（自动注入会导致组件与 v-for 运作紧密耦合，不易后续场合的重复使用），因为组件有自己独立的作用域，我们可以通过组件 prop 的方式来将数据传递到组件内
